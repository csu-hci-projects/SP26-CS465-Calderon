# AirDesk Paper Workspace

This folder holds the CS465 paper and delivery materials for AirDesk.

AirDesk should be written as a system paper with small pilot evidence. The paper should be honest about the scope: the prototype is an OS-level spatial input layer for Hyprland, and the evaluation will likely be a small pilot with Caden and one roommate rather than a broad user study.

## Folder Map

- `latex-source/` - ACM manuscript LaTeX source and local template files.
- `drafts/` - outlines, section drafts, and narrative planning.
- `research/` - related-work notes, source tracker, and reading synthesis.
- `findings/` - current prototype evidence, pilot plans, results notes, and limitations.
- `PDFs-LiteratureSurvey/` - PDFs for cited research papers, excluding books/websites.
- `assets/figures/` - diagrams, screenshots, and paper figures.
- `videos/` - video planning notes and links for the final README.
- `final-package/` - submission checklist and eventual zip/README planning.

## Current Paper Position

Working title:

> AirDesk: Mid-Air Gestures for Situationally Impaired Desktop Interaction

Current thesis:

> Mid-air gestures are unlikely to replace keyboard and mouse for precision desktop work, but may be useful as a secondary command channel for coarse, spatial, low-text actions when conventional input is inconvenient, unavailable, dirty, painful, or physically costly.
